# Control UI 左侧主菜单说明

适用于 `http://127.0.0.1:19001/chat`（或带 `basePath` 的 Control UI）。下表说明每个菜单的**功能、相关配置、典型默认、最佳选择**。

| 菜单（分组）                    | 功能                                                                                      | 相关配置                                                                                                                 | 默认                                                                            | 最佳选择                                                                                |
| ------------------------------- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **聊天** (Chat)                 | 与 Gateway 直连的对话；发消息、看流式回复与工具卡片，用于快速干预与调试。                 | 无单独配置；会话用 `sessionKey`（如 `agent:dev:main`），模型/会话策略见 `agents.*`、`session.*`。                        | 默认会话为配置中的 main 会话（如 `agent:dev:main`）。                           | 日常对话与技能测试用聊天；敏感操作在可信环境使用。                                      |
| **控制 → 概览** (Overview)      | 网关连接状态、入口点、健康快照；展示 WebSocket URL、Token、默认会话键、语言等。           | `gateway.controlUi.basePath`、`gateway.auth.mode` / `gateway.auth.token`、`session.mainKey`。                            | 端口 18789；auth 需 token 或 password；本地 127.0.0.1 可自动通过配对。          | 首次打开在此填 Token 并连接；远程用 Tailscale Serve 或绑定 tailnet + token。            |
| **控制 → 频道** (Channels)      | 管理各消息频道（WhatsApp/Telegram/Discord/Slack 等）状态、扫码登录、每频道配置。          | `channels.<provider>.enabled`、`channels.<provider>.dmPolicy`、`allowFrom`、`groupPolicy`、各频道 botToken/credentials。 | 各频道默认未启用；`dmPolicy` 常用 `pairing`。                                   | 按需启用频道；生产环境用 `allowlist` 或 `pairing`，慎用 `open`。                        |
| **控制 → 实例** (Instances)     | 查看已连接客户端与节点的在线信号（presence beacons）。                                    | 无独立配置；由 Gateway 与节点注册决定。                                                                                  | 无。                                                                            | 用于确认多端/多节点是否在线，排查“收不到消息”时先看此处。                               |
| **控制 → 会话** (Sessions)      | 查看活动会话列表，按会话设置 thinking/verbose 等覆盖。                                    | `session.dmScope`、`session.threadBindings`、`session.reset`；会话级覆盖由 UI 写入会话存储。                             | `dmScope` 等见 configuration；单会话默认用配置的 main 会话。                    | 多用户时用 `per-channel-peer`；需按会话调参时在此改。                                   |
| **控制 → 使用情况** (Usage)     | 监控 API 用量与成本（按 provider/模型等）。                                               | 用量数据来自 Gateway 统计；部分 provider 需配置 usage 上报。                                                             | 无配置则仅本地统计。                                                            | 定期查看以控制成本；配合 `agents.defaults.models` 与模型切换使用。                      |
| **控制 → 定时任务** (Cron Jobs) | 管理 cron：添加/编辑/启用/禁用、运行历史；支持主会话与独立 agent、公告/Webhook/无推送等。 | `cron.jobs`、每项 `schedule`/`delivery`/`agentId` 等；`cron.webhookToken` 可选。                                         | 无预置任务。                                                                    | 提醒类用主会话 + 公告；独立任务用 isolated agent；Webhook 需填有效 URL。                |
| **代理 → 代理** (Agents)        | 管理 agent 工作区、工具策略与身份（workspace、tools、identity）。                         | `agents.defaults`、`agents.list`（含 `id`、`workspace`、`model`、`identity`、`tools`）。                                 | 单 agent（如 dev）、默认 workspace 为 `~/.openclaw/workspace`。                 | 多 agent 时为每个配好 workspace 与 identity；工具用 `tools.profile` + allow/deny 收紧。 |
| **代理 → 技能** (Skills)        | 管理技能可用性、API Key 注入、安装状态。                                                  | `skills.entries.<key>.enabled`、`skills.entries.<key>.env`/`apiKey`、`skills.allowBundled`、`skills.load.extraDirs`。    | 无 `allowBundled` 时所有 bundled 符合条件即加载；未配 `enabled: false` 即启用。 | 按需用 `allowBundled` 限制内置技能；第三方技能用 `skills.entries` 关掉或注入 Key。      |
| **代理 → 节点** (Nodes)         | 配对设备、能力与命令暴露（如 Pi/节点列表）。                                              | 节点侧配置 + Gateway 的节点注册；exec 审批等见 `tools.elevated.*`。                                                      | 无预置节点。                                                                    | 需要远程/多机时在此确认节点已配对、能力可见。                                           |
| **设置 → 配置** (Config)        | 安全编辑 `~/.openclaw/openclaw.json`（Form + Raw），应用并可选重启。                      | 即整份 openclaw.json；路径可由 `OPENCLAW_CONFIG_PATH` 指定。                                                             | 文件不存在则用内存默认；保存后热加载（部分变更需重启）。                        | 改完用 Form 的 Apply 或 Raw 保存；大改前备份；用 base-hash 避免并发覆盖。               |
| **设置 → 调试** (Debug)         | 网关快照、健康、模型列表、事件日志、手动 RPC 调用。                                       | 只读；展示当前 `status`/`health`/`models.list` 等。                                                                      | 无。                                                                            | 排查连接/模型/事件时用；生产环境限制访问。                                              |
| **设置 → 日志** (Logs)          | 网关文件日志的实时 tail，支持过滤与导出。                                                 | 日志路径与级别由 `logging.*` 等决定。                                                                                    | 默认写文件，具体路径见 docs。                                                   | 问题定位时开实时 tail；注意日志中可能含敏感信息。                                       |

---

**说明**

- **菜单与路由**：聊天 = `/chat`，控制 = `/overview`、`/channels`、`/instances`、`/sessions`、`/usage`、`/cron`，代理 = `/agents`、`/skills`、`/nodes`，设置 = `/config`、`/debug`、`/logs`。带 basePath 时上述路径前加 basePath。
- **默认会话**：URL 中 `session=agent%3Adev%3Amain` 表示会话键 `agent:dev:main`（agentId=dev，main 会话）；默认会话键可在概览里改。
- **更多**：完整配置见 [Configuration](https://docs.openclaw.ai/gateway/configuration)、[Configuration reference](https://docs.openclaw.ai/gateway/configuration-reference)；Control UI 行为见 [Control UI](https://docs.openclaw.ai/web/control-ui)。
